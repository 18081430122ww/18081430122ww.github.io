hexo创建博客遇到的一些问题

处理内存不足

在生成过程中遇到这个错误：

严重错误：CALL_AND_RETRY_LAST分配失败-内存不足

通过更改hexo-cli（which hexo以查找文件）的第一行来增加Node.js堆内存的大小。

＃！/ usr / bin / env节点–max_old_space_size = 8192

没有找到这样的文件或者目录

Error: ENOENT: no such file or directory
则可能是由于在标签，类别或文件名中混用了大写和小写字母。Git无法自动合并此更改，因此会中断自动分支。

解决这个问题可以尝试：

1.检查每个标签和类别的大小写，并确保它们相同。
2.承诺
3.清理并构建： ./node_modules/.bin/hexo clean && ./node_modules/.bin/hexo generate
4.手动将公用文件夹复制到您的桌面
5.将分支从主分支切换到本地部署分支
6.将公用文件夹的内容从您的桌面复制到Deployment分支中
7.承诺。您应该看到出现任何可以手动解决的合并冲突。
8.切换回您的master分支并正常部署： ./node_modules/.bin/hexo deploy
